
# Version 6.1 - Push Notification & Password Reset Fixes

## Issues Fixed

### 1. Push Notification Issue for Admin ✅

**Problem Identified:**
The admin user (lydonmn@gmail.com) has:
- ✅ Notifications enabled (`daily_report_notifications: true`)
- ✅ Opted in to Folly Beach reports (`notification_locations: ["folly-beach"]`)
- ❌ **No push token** (`push_token: null`)

The notification edge function filters by `push_token IS NOT NULL`, so users without tokens don't receive notifications.

**Root Cause:**
Push tokens can only be obtained on:
- Physical iOS/Android devices (not simulators)
- Apps built with EAS Build (not Expo Go)

The admin is likely testing on a simulator or using Expo Go, which cannot register for push notifications.

**Solution Implemented:**
1. ✅ Updated `send-daily-report-notifications` edge function with:
   - Better logging to show which users don't have tokens
   - Detailed statistics in response
   - Graceful handling of missing tokens
   - Filter by `notification_locations` array

2. ✅ Added `PushNotificationDebugger` component for admin panel:
   - Run diagnostics on any user
   - Check push token status
   - Send test notifications
   - View notification settings

3. ✅ Enhanced notification status display in admin-users screen:
   - Shows if notifications are enabled
   - Shows if push token is registered
   - Color-coded status indicators

**How to Fix for Admin:**
The admin needs to:
1. Build the app with EAS Build: `eas build --platform ios --profile preview`
2. Install on a physical iPhone
3. Sign in to the app
4. Go to Profile → Enable "Daily Report Notifications"
5. The app will automatically register the push token
6. Notifications will start working at 5 AM EST

### 2. Forgot Password Flow ✅

**Problem:**
No way for users to reset their password if they forgot it.

**Solution Implemented:**
1. ✅ Added "Forgot Password?" link on login screen
2. ✅ Created password reset email flow
3. ✅ Created `/reset-password` screen to handle password updates
4. ✅ Added deep link support with `surfvista://` scheme
5. ✅ Integrated with Supabase Auth password reset

**User Flow:**
1. User clicks "Forgot Password?" on login screen
2. Enters their email address
3. Receives password reset email from Supabase
4. Clicks link in email → Opens app to `/reset-password` screen
5. Enters new password (min 6 characters)
6. Confirms new password
7. Password is updated
8. User is redirected to login screen
9. User signs in with new password

**Technical Implementation:**
- Uses `supabase.auth.resetPasswordForEmail()` to send reset email
- Deep link: `surfvista://reset-password`
- Uses `supabase.auth.updateUser({ password })` to update password
- Validates password strength (min 6 characters)
- Confirms passwords match before updating
- Clears recovery session after successful reset

## Important Notes

- **Push tokens are device-specific** - Each device gets its own token
- **Simulators cannot receive push notifications** - Must use physical devices
- **Expo Go cannot receive push notifications** - Must use EAS Build
- **Tokens expire** - Users may need to re-enable notifications if token expires
- **Password reset links expire** - Users should use them within 1 hour
