
# Push Notifications on TestFlight - Complete Guide

## ✅ What We Fixed in V6.9

```json
"extra": {
  "eas": {
    "projectId": "e1ee166c-212b-4eca-a1d7-44183b7be073"
  }

```
**Why this matters:** Without the EAS project ID, Expo cannot generate valid push tokens on physical devices and TestFlight builds. This was the PRIMARY reason notifications weren't working.

### 2. **Automatic Push Token Registration on Login**
- Push tokens are now automatically registered when users sign in
- If a user has notifications enabled but no valid token, the system will attempt to register one
- This ensures admin and all users get tokens without manual intervention

### 3. **Enhanced Logging Throughout**
- Detailed logs in `utils/pushNotifications.ts` show exactly what's happening
- Edge function logs show which users have valid tokens and why some don't
- Easy to diagnose issues by checking logs

## 🔍 How Push Notifications Work

### For Development (Expo Go)
- Push notifications **DO NOT WORK** in Expo Go
- You'll see "EXPERIENCE_NOT_FOUND" errors - this is EXPECTED
- The app will save your notification preferences but won't get a real token

### For TestFlight/App Store Builds
- Push notifications **WORK PERFECTLY** when built with EAS Build
- The EAS project ID allows Expo to generate valid push tokens
- Users on physical devices will receive notifications at 5 AM EST

## 📱 Testing Push Notifications on TestFlight

### Step 1: Ensure App is Built with EAS
```bash
# Build for TestFlight
eas build --platform ios --profile production-ios

# Submit to TestFlight
eas submit --platform ios
```

### Step 2: Install from TestFlight
- Download the app from TestFlight (NOT Expo Go)
- Sign in with your account
- The app will automatically attempt to register a push token

### Step 3: Enable Notifications
1. Go to Profile tab
2. Toggle "Daily Surf Report" to ON
3. When prompted, tap "Allow" for notifications
4. If you previously denied, tap "Open Settings" and enable notifications

### Step 4: Verify Token Registration
- Check the Debug Info section at the bottom of the Profile screen
- Look for "Permission Status: Granted ✓"
- The system will automatically register your push token

### Step 5: Test Notification Sending
- As admin, go to Admin Data screen
- Tap "Pull Data & Generate Reports"
- The system will automatically send notifications to all eligible users

## 🐛 Troubleshooting

### Issue: "No valid push token"
**Cause:** User is running in Expo Go or simulator
**Solution:** Build with EAS and install on physical device or TestFlight

### Issue: "Permission Status: Denied"
**Cause:** User denied notification permissions
**Solution:** 
1. Go to iPhone Settings
2. Scroll to SurfVista
3. Tap Notifications
4. Enable "Allow Notifications"
5. Return to app and toggle notifications again

### Issue: "EXPERIENCE_NOT_FOUND" error
**Cause:** Running in development without EAS build
**Solution:** This is expected in development. Build with EAS for production.

### Issue: Admin not receiving notifications
**Checklist:**
1. ✅ App installed from TestFlight (not Expo Go)
2. ✅ Notifications enabled in Profile tab
3. ✅ iOS notification permissions granted
4. ✅ Push token registered (check Debug Info)
5. ✅ Location selected in notification settings
6. ✅ Daily report generated for that location

## 🔐 Required iOS Permissions

The app.json already includes:
```json
"UIBackgroundModes": [
  "remote-notification",
  "fetch",
  "processing"
]
```

This allows the app to:
- Receive push notifications in the background
- Wake up to process notifications
- Display notifications even when app is closed

## 📊 Monitoring Notifications

### Check Edge Function Logs
```bash

supabase functions logs send-daily-report-notifications
```

Look for:
- "Total opted-in users: X"
- "Eligible users (with valid tokens): Y"
- "Users without valid tokens: Z"
- Individual user token status

### Check Frontend Logs
- Open the app
- Check console logs for push token registration
- Look for "[Push Notifications] ✅ Token obtained: ExponentPushToken[...]"

## ✅ Success Criteria

You'll know push notifications are working when:

1. **Profile Screen Shows:**
   - Permission Status: Granted ✓
   - Daily Notifications: Enabled
   - Debug Info shows a valid push token (starts with "ExponentPushToken[")

2. **Edge Function Logs Show:**
   - "Eligible users (with valid tokens): 1" (or more)
   - "✅ Sent to [your-email]"
   - "Successfully sent: 1" (or more)

3. **You Receive:**
   - Push notification at 5 AM EST
   - Notification shows surf report summary
   - Tapping notification opens the Report tab
